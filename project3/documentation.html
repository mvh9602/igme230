TOWER Documentation

HOW TO PLAY

  • Click to drop a box
  • After landing, boxes will trim to fit box below it
  • Each spawned box has the size of the last box you dropped
  • Dropping a box and missing the last one you dropped will end the game
  • Dropping a box that becomes too small (smaller than a square) will end the game
  • Higher stacks with smaller boxes add up to higher scores
  • Build the highest tower with the highest score you can!

DEV PROCESS

  • Started by researching possible libraries
  • Had difficulty finding one that suited my needs and still looked good, settled on Matter.js
  • Looked pretty far into Matter.js to minimize confusion while building the game
  • Created html and css framework for the page
  • Got a base for working in Matter.js off of the website
  • Started adapting demos until I could spawn rectangles that moved back and forth before falling
  • Got rectangles to become static after landing
  • Rectangles only spawn so often and fall straight
  • Researched how to work with collisions (website documentation is highly lacking)
  • After a lot of time, rectangles lost hanging space after colliding, and I started to actually have a game
  • Added a very hacky reset
  • Improved visuals (somewhat) and added a score algorithm
  • Made it so you can lose the game by dropping a box off the edge or making boxes too small
  • Allowed game to continue forever by adding scrolling and a gradual increase to speed of the game
  • Celebrated finally finishing the game by commenting everything
  • Worked on documentation and finished up css
